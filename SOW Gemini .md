## 1. วัตถุประสงค์การจัดทำ Statement of Work
เอกสารฉบับนี้จัดทำขึ้นเพื่อกำหนดขอบเขตการดำเนินงาน เงื่อนไข และข้อตกลงในการพัฒนาระบบบริหารจัดการร้านตัดผม ระหว่างผู้ว่าจ้างและผู้รับจ้าง เพื่อให้การดำเนินโครงการครอบคลุมทั้งส่วนงานหน้าร้าน (POS) และระบบออนไลน์ (Booking) เป็นไปอย่างมีประสิทธิภาพและบรรลุเป้าหมายทางธุรกิจ

## 2. วัตถุประสงค์โครงการ

| วัตถุประสงค์ | ตัวชี้วัดความสำเร็จ |
|------------|-------------------|
| 1. เพิ่มประสิทธิภาพการบริหารคิว | - ลดเวลารอคิวของลูกค้าลง 30%<br>- ลดอัตราการทิ้งคิว (No Show) เหลือต่ำกว่า 5% |
| 2. ยกระดับการบริหารความสัมพันธ์ลูกค้า | - เพิ่มอัตราลูกค้ากลับมาใช้ซ้ำ (Retention) 20%<br>- มีฐานข้อมูลประวัติทรงผมลูกค้าครบ 100% |
| 3. เพิ่มความแม่นยำทางบัญชีและสต็อก | - ลดข้อผิดพลาดการคิดเงินและค่าคอมมิชชั่นเหลือ 0%<br>- ลดอัตราสินค้าสูญหาย/หมดอายุลง 50% |
| 4. บริหารจัดการประสิทธิภาพช่าง | - เพิ่มรายได้เฉลี่ยต่อช่าง 15% จากการจัดตารางงานที่มีประสิทธิภาพ<br>- คะแนนความพึงพอใจต่อช่าง > 4.5/5 |

## 3. ขอบเขตของโครงการ

### 3.1 ขอบเขตการดำเนินงาน

ระบบงานหลักประกอบด้วย:

| โมดูล | ฟังก์ชันการทำงาน | ผู้ใช้งาน |
|-------|-----------------|-----------|
| 1. ระบบจองคิวและจัดการคิว | - จองออนไลน์/หน้าร้าน (Walk-in)<br>- จอแสดงสถานะคิว<br>- การแจ้งเตือนคิว | - ลูกค้า<br>- พนักงานต้อนรับ |
| 2. ระบบลูกค้าสัมพันธ์ (CRM) | - จัดเก็บประวัติทรงผม (รูปถ่าย)<br>- ระบบสมาชิกและสะสมแต้ม<br>- ระดับสมาชิก (Tier) | - ลูกค้า<br>- ช่างตัดผม |
| 3. ระบบปฏิบัติการและ POS | - ชำระเงินและออกใบเสร็จ<br>- คำนวณค่าคอมมิชชั่นช่าง<br>- ตัดสต็อกสินค้า | - แคชเชียร์<br>- ผู้จัดการร้าน |
| 4. ระบบบริหารงานและรายงาน | - จัดการตารางงานช่าง<br>- Dashboard ผู้บริหาร<br>- รายงานรายได้และประสิทธิภาพ | - ผู้บริหาร<br>- เจ้าของร้าน |

### 3.2 ระยะเวลา
โครงการมีระยะเวลาดำเนินการทั้งสิ้น 5 เดือน ตั้งแต่วันที่ 1 มกราคม 2025 ถึง 31 พฤษภาคม 2025

### 3.3 งวดงานและงวดเงิน

| งวดที่ | ระยะเวลา | ขอบเขตงาน | เงื่อนไขการส่งมอบ | สัดส่วนการจ่ายเงิน |
|-------|----------|------------|------------------|------------------|
| 1 | 1.5 เดือน | - วิเคราะห์ความต้องการ (Requirement)<br>- ออกแบบ UX/UI<br>- ออกแบบฐานข้อมูล | - เอกสาร SRS และ SDD<br>- Prototype หน้าจอ UI | 30% |
| 2 | 2 เดือน | - พัฒนาระบบจองคิว (Booking)<br>- พัฒนาระบบ CRM และ Member<br>- พัฒนาระบบ POS และ Inventory | - ระบบเวอร์ชัน Alpha<br>- ผลการทดสอบ Unit Test | 40% |
| 3 | 1.5 เดือน | - พัฒนาระบบรายงานและ Dashboard<br>- ทดสอบระบบรวม (UAT)<br>- ติดตั้งและอบรมพนักงาน | - ระบบพร้อมใช้งาน (Production)<br>- คู่มือและ Source Code | 30% |

### 3.4 การรายงานความคืบหน้าโครงการ
การรายงานความคืบหน้าจะดำเนินการดังนี้:
1. รายงานประจำสัปดาห์: ส่งทุกวันศุกร์ เวลา 17:00 น.
2. การประชุม Sprint Review: จัดทุก 2 สัปดาห์เพื่อดูความคืบหน้าของฟีเจอร์
3. รายงานสรุปประจำงวดงาน: ส่งเมื่อส่งมอบงานแต่ละงวด

### 3.5 ปัจจัยที่มีผลกระทบต่อโครงการ

| ปัจจัย | ผลกระทบ | มาตรการรองรับ |
|-------|----------|---------------|
| ความเสถียรของอินเทอร์เน็ต | - ระบบจองและ POS ใช้งานไม่ได้ | - ออกแบบระบบ Offline Mode ให้ทำงานพื้นฐานได้ |
| ความถนัดด้าน IT ของช่าง | - การบันทึกข้อมูลล่าช้า<br>- ข้อมูลไม่ครบถ้วน | - ออกแบบ UI ให้ใช้ง่ายที่สุด<br>- อบรมเข้มข้นเน้นปฏิบัติจริง |
| อุปกรณ์ Hardware | - เครื่อง POS/Tablet ไม่รองรับ | - กำหนด Spec อุปกรณ์ให้ชัดเจนก่อนเริ่มพัฒนา |

## 4. ข้อตกลงร่วมกัน

| หัวข้อ | รายละเอียด |
|-------|------------|
| การรับประกัน | - ระยะเวลา 1 ปี นับจากวันส่งมอบงวดสุดท้าย<br>- แก้ไข Bug และข้อผิดพลาดของระบบฟรี |
| SLA (Service Level Agreement) | - การันตีความพร้อมใช้งาน (Uptime) 99.5%<br>- แก้ไขปัญหาร้ายแรง (Critical Issue) ภายใน 4 ชั่วโมง |
| การสนับสนุน | - บริการ Remote Support จันทร์-อาทิตย์ (09:00 - 20:00)<br>- มีคู่มือการใช้งานรูปแบบ Video Clip |

## 5. การบริหารการเปลี่ยนแปลง
การบริหารการเปลี่ยนแปลงจะดำเนินการผ่าน Change Control Board (CCB) โดยมีขั้นตอนดังนี้:
1. ผู้ขอเปลี่ยนแปลงกรอกแบบฟอร์ม Change Request (CR)
2. ทีมพัฒนาประเมินผลกระทบต่อเวลาและต้นทุน
3. คณะกรรมการพิจารณาอนุมัติ/ไม่อนุมัติ
4. หากอนุมัติ จะมีการปรับปรุงแผนงานและเอกสารที่เกี่ยวข้อง
